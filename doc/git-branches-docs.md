# Документация по веткам Git

Этот документ описывает назначение и текущий статус веток в репозитории.

## Основные ветки

### `main`
- **Назначение:** Основная ветка. Содержит стабильный, готовый к релизу код. Прямые коммиты в `main` запрещены.
- **Слияние:** Обновляется только путем слияния ветки `dev` после полного тестирования.

### `dev`
- **Назначение:** Предрелизная ветка. Агрегирует все готовые `feature/*` ветки для комплексного тестирования перед релизом.
- **Статус:** Активна. Является основной для разработки.
- **Последнее изменение:** Добавлен подробный анализ всего проекта (Frontend + Backend) в `doc/project.md`.

## Ветки функционала (`feature/*`)

Здесь будет документироваться каждая новая ветка, созданная для реализации определенного блока задач.

### `feature/redesign`
- **Назначение:** Небольшой редизайн сайта. Изменение цветовой палитры и добавление "живых" эффектов.
- **Задачи:** Изменить основной цвет с розового на фиолетовый. Добавить эффект "Liquid Glass" на некоторые элементы интерфейса. Отчёт о багах, возникших после внедрения частиц. Исправление багов с кликабельностью, Z-index и просвечиванием. Откат фона секции "Преимущества подписки" на главной странице. Реализация страницы "Эксклюзивы", кастомного аудиоплеера и логики скачивания файлов для соответствующих тарифов. Реализация страницы "О нас".
- **Статус:** `active` (все известные баги устранены, внесены правки по запросу, реализован эксклюзивный контент, реализована страница "О нас").

### `feature/account-page`
- **Назначение:** Улучшение страницы личного кабинета для корректного отображения статуса подписки и реализации симулированной отмены подписки.
- **Задачи:** P1.1 (AccountPage subscription management).
- **Статус:** Завершена и слита в `dev`.

### `feature/subscription-purchase`
- **Назначение:** Реализация имитации покупки подписки на бэкенде и интеграция ее с фронтендом на странице `PricingPage`.
- **Задачи:** P1.1 (backend simulated purchase, frontend integration on PricingPage).
- **Статус:** Завершена и слита в `dev`.

### `feature/blog-posts-api`
- **Назначение:** Реализация бэкенда API для блог-постов и интеграция их с фронтендом.
- **Задачи:** P0.2 (posts API), P0.3 (integration of posts API on frontend).
- **Статус:** Завершена и слита в `dev`.

### `feature/auth-backend`
- **Назначение:** Реализация бэкенда для аутентификации (регистрация, логин, профиль) и интеграция API релизов и артистов с фронтендом.
- **Задачи:** P0.2 (auth, releases, artists API), P0.3 (integration of auth, releases, artists API on frontend), P1.1 (integration of auth forms).
- **Статус:** Завершена и слита в `dev`.

### `feature/subscription-system`
- **Назначение:** Реализация полного цикла системы подписок.
- **Задачи:** P1.1 (создание `PricingPage`, `AccountPage`, интеграция платежей), P2.1, P2.2 (механизм Paywall).
- **Статус:** Завершена и слита в `dev`.

### `feature/design-system-audit`
- **Назначение:** Проведение аудита всех компонентов и страниц на соответствие дизайн-системе.
- **Задачи:** P2.3 (Полное соответствие дизайн-системе).
### `feature/responsive-design-audit`
- **Назначение:** Проверка и доработка адаптивной верстки для всех страниц на всех брейкпоинтах.
- **Задачи:** P2.4 (Адаптивность).
### `feature/header-footer-logic`
- **Назначение:** Реализация Header и Footer с корректными ссылками и логикой отображения для авторизованных/неавторизованных пользователей.
- **Задачи:** P2.5 (Навигация).
### `feature/polls-backend`
- **Назначение:** Реализация бэкенда для создания голосований и сбора голосов.
### `feature/polls-frontend`
- **Назначение:** Создание UI для отображения активных и завершенных голосований с результатами (PollsPage.tsx).
- **Задачи:** P3.1.2 (Система голосований - фронтенд).
### `feature/pro-library-storage`
- **Назначение:** Настройка файлового хранилища (S3/Yandex Object Storage) для профессиональных материалов (ProLibraryPage.tsx).
### `feature/pro-library-frontend`
- **Назначение:** Реализация UI для каталога с фильтрацией и скачиванием (ProLibraryPage.LibraryPage.tsx).
- **Задачи:** P3.2.2 (PRO-библиотека - фронтенд).
### `feature/submit-demo-backend`
- **Назначение:** Реализация бэкенда для загрузки файлов и отслеживания статусов (SubmitDemoPage.tsx).
- **Задачи:** P3.3.1 (Модуль отправки демо - бэкенд).
### `feature/submit-demo-frontend`
- **Назначение:** Создание UI с формой отправки и таблицей статусов "Мои демо" (SubmitDemoPage.tsx).
- **Задачи:** P3.3.2 (Модуль отправки демо - фронтенд).
- **Статус:** `merged`
